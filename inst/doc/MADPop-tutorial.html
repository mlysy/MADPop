Bayesian Testing of Equal Genotypic Proportions between Populations

Wookjung P. Kim, Terin Robinson, Martin Lysy

2016-11-19

This tutorial shows how to compute Bayesian posterior p-values for the usual chi-squared (\(\mathcal X\)) and likelihood ratio (\(\Lambda\)) statistics under a hierarchical model.

require(MADPop)

This data is converted into a numeric format to be used with MADPop with the command UM.suff.

UM.suff is a function that converts a matrix of raw allele data into the quantities needed to describe the corresponding Hardy-Weinberg model. The function will return a list with the following:

  • A: Vector of unique alleles names. The allele numbers in the following quantities correspond to the indices of A.
  • G: 4-column matrix of unique genotype combinations. The presence of 0’s indicates that less than four alleles were amplified indicating that a given genotype either has less than 4 distinct alleles or that some alleles are duplicated.
  • tab: Observed data in a simplified numerical format. This is a contingency table with rows given by the unique elements of popID and columns given by each row of G.
Xsuff <- UM.suff(fish215)

Xsuff is a list with three components:

  • A: The allele names found in the data frame fish215. Each is encoded by a number in the given order.

Model and Null Hypothesis

Let \(Y_k\) denote the unique genotype categories… The model is \[ Y_k | \rho_k \stackrel{\textrm{ind}}{\sim} \textrm{Multinomial}(\rho_k, N_k), \qquad \rho_k \stackrel{\textrm{iid}}{\sim} \textrm{Dirichlet}(\alpha), \qquad (\bar \alpha, \alpha_0) \sim \frac{1}{(1 + \alpha_0)^2},\] where \(\alpha_0 = \sum_{j=1}^C \alpha_j\) and \(\bar \alpha = \alpha/\alpha_0\).

Posterior p-value Calculation

Let \(T\) denote any test statistic for evaluating the null hypothesis that lakes Michipicoten and Simcoe have the same genotype proportions. Two of the most popular statistics to this end are:

  • Pearson’s chi-squared statistic: \[ \mathcal X = ??? \]
  • The likelihood ratio statistic: \[ \Lambda = ??? \]

With \(T_{\textrm{obs}}\) being the observed value of the test statistic in the data at hand, the posterior p-value is defined as \[ p_{\textrm{post}} = \textrm{Pr}(T > T_{\textrm{obs}} | data, H_0). \] This is calculated in MADPop by using rho.post to generate samples from \(p(T | data, H_0)\):

# number of observations in each lake
N1 <- sum(Xsuff$tab["Michipicoten",])
N2 <- sum(Xsuff$tab["Simcoe",])
# generate samples from p(T | data, H_0)
T.post <- UM.eqtest(N1 = N1, N2 = N2, p0 = rho.post, nreps = 1e4)
## bootstrap sample 5000/10000
## bootstrap sample 10000/10000
head(T.post) # computes both chi^2 and LRT
##          chi2      LRT
## [1,] 23.20000 31.40653
## [2,] 22.00000 29.76468
## [3,] 31.33333 43.31492
## [4,] 22.00000 30.13158
## [5,] 25.41905 33.75421
## [6,] 24.53333 33.44532
# calculate T.obs
tab.obs <- Xsuff$tab[lnames,] # observed table
# automatically removes categories where both lakes have zero observations
T.obs <- c(chi2 = chi2.stat(tab.obs),
           LRT = LRT.stat(tab.obs))
p.post <- c(chi2 = mean(T.post[,"chi2"] >= T.obs["chi2"]),
            LRT = mean(T.post[,"LRT"] >= T.obs["LRT"]))
p.post # posterior p-values
##   chi2    LRT 
## 0.3426 0.3337
